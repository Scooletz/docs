# Session: Querying: How to Use Search
<LanguageSwitcher supportedLanguages={supportedLanguages} />
<LanguageContent language="csharp">


More complex text searching can be achieved by using the `Search` extension method. This method allows you to pass one or more search terms that will be used in the searching process for a particular field (or fields).

## Syntax

<TabItem value="search_1" label="search_1">
<CodeBlock language="csharp">
{`IRavenQueryable<T> Search<T>(
        Expression<Func<T, object>> fieldSelector,
        string searchTerms,
        decimal boost = 1,
        SearchOptions options = SearchOptions.Guess,
        SearchOperator @operator = SearchOperator.Or)
`}
</CodeBlock>
</TabItem>

| Parameters | | |
| ------------- | ------------- | ----- |
| **fieldSelector** | Expression&lt;Func&lt;TResult&gt;&gt; | Points index field that should be used for querying. |
| **searchTerms** | string | Space separated terms e.g. 'John Adam' means that we will look in selected field for 'John' or 'Adam'. Wildcards can be specified. |
| **boost** | decimal | Boost value. Default: `1`. |
| **options** | SearchOptions | Explicitly set relation between each Search function. One of the following: `Or`, `And`, `Not`, `Guess`. Default: `SearchOptions.Guess`. |

## Example I - Dynamic Query

<Tabs groupId='languageSyntax'>
<TabItem value="Sync" label="Sync">
<CodeBlock language="csharp">
{`List<User> users = session
    .Query<User>()
    .Search(u => u.Name, "a*")
    .ToList();
`}
</CodeBlock>
</TabItem>
<TabItem value="Async" label="Async">
<CodeBlock language="csharp">
{`List<User> users = await asyncSession
    .Query<User>()
    .Search(u => u.Name, "a*")
    .ToListAsync();
`}
</CodeBlock>
</TabItem>
<TabItem value="RQL" label="RQL">
<CodeBlock language="sql">
{`from Users 
where search(Name, 'a*')
`}
</CodeBlock>
</TabItem>
</Tabs>

## Example II - Query Using Static Index

<Tabs groupId='languageSyntax'>
<TabItem value="Sync" label="Sync">

    .Search(x => x.Hobbies, "sport")
    .ToList();
`}
</CodeBlock>
</TabItem>
<TabItem value="Async" label="Async">
<CodeBlock language="csharp">

    .Query<User, Users_ByNameAndHobbies>()
    .Search(x => x.Name, "Adam")

    .ToListAsync();
`}
</CodeBlock>
</TabItem>

`}
</CodeBlock>
</TabItem>
</Tabs>

## Example III - Boosting Usage

<Tabs groupId='languageSyntax'>
<TabItem value="Sync" label="Sync">
<CodeBlock language="csharp">
{`List<User> users = session
    .Query<User>("Users/ByHobbies")
    .Search(x => x.Hobbies, "I love sport", boost: 10)
    .Search(x => x.Hobbies, "but also like reading books", boost: 5)
    .ToList();
`}
</CodeBlock>
</TabItem>
<TabItem value="Async" label="Async">
<CodeBlock language="csharp">
{`List<User> users = await asyncSession
    .Query<User>("Users/ByHobbies")
    .Search(x => x.Hobbies, "I love sport", boost: 10)
    .Search(x => x.Hobbies, "but also like reading books", boost: 5)
    .ToListAsync();
`}
</CodeBlock>
</TabItem>
<TabItem value="RQL" label="RQL">
<CodeBlock language="sql">
{`from index 'Users/ByHobbies' 
where boost(search(Hobbies, 'I love sport'), 10) or boost(search(Hobbies, 'but also like reading books'), 5)
`}
</CodeBlock>
</TabItem>
</Tabs>

<Admonition type="note" title="">
To leverage the searching capabilities with the usage of static indexes, please remember to enable full-text search in field settings of the index definition.
</Admonition>

More complex text searching can be achieved by using the `search()` method. This method allows you to pass one or more search terms that will be used in the searching process for a particular field (or fields).

## Syntax

<TabItem value="search_1" label="search_1">
<CodeBlock language="js">
{`query.search(fieldName, searchTerms);
query.search(fieldName, searchTerms, operator);
query.boost(boost);
`}
</CodeBlock>
</TabItem>

| Parameters | | |
| ------------- | ------------- | ----- |
| **fieldName** | string | Points index field that should be used for querying. |
| **searchTerms** | string | Space separated terms e.g. 'John Adam' means that we will look in selected field for 'John' or 'Adam'. Wildcards can be specified. |
| **boost** | number | Boost value. Default: `1`. |
| **operator** | `SearchOperator` | Explicitly set relation between each Search function. One of the following: `"OR"`, `"AND"`. |
