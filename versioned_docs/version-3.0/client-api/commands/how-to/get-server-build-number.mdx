# Commands: How to get server build number?
<LanguageSwitcher supportedLanguages={supportedLanguages} />
<LanguageContent language="csharp">


To check with what version of server commands you are working use `GetBuildNumber` command from `GlobalAdmin`.

## Syntax

<TabItem value="build_number_1" label="build_number_1">
<CodeBlock language="csharp">
{`BuildNumber GetBuildNumber();
`}
</CodeBlock>
</TabItem>

---

<TabItem value="buildnumber" label="buildnumber">
<CodeBlock language="csharp">
{`public class BuildNumber
\{
	public string ProductVersion \{ get; set; \}

	public string BuildVersion \{ get; set; \}
\}
`}
</CodeBlock>
</TabItem>

| Return Value | BuildNumber | Information about current product (server) version and build number. |
| ------------- | ------------- | ----- |
| **ProductVersion** | string | String representing current product version e.g. `"3.0.0 / 6dce79a"` |
| **BuildVersion** | string | String indicating current build version e.g. `"3260"` |

## Example

<TabItem value="build_number_2" label="build_number_2">
<CodeBlock language="csharp">
{`BuildNumber buildNumber = store
	.DatabaseCommands
	.GlobalAdmin
	.GetBuildNumber();
`}
</CodeBlock>
</TabItem>

## Remarks

`BuildVersion` for custom builds is `"13"`

</LanguageContent>

<LanguageContent language="http">


To check with what version of server you are working use `build/version` endpoint.

## Syntax

<TabItem value="json" label="json">
<CodeBlock language="json">
{`curl \\
	http://\{serverName\}/databases/\{databaseName\}/build/version \\
	-X GET 
`}
</CodeBlock>
</TabItem>

### Response

| Status code | Description |
| ----------- | - |
| `200` | OK |

| Return Value | Description |
| ------------- | ------------- |
| **ProductVersion** | String representing current product version e.g. `"3.0.0 / 6dce79a"` |
| **BuildVersion** |  String indicating current build version e.g. `"3260"` |
