We can clear all session operations and stop tracking of all entities by using [Clear](../../../client-api/session/how-to/clear-a-session.mdx) method, but sometimes there is need to only to do a cleanup only for one entity. For this purpose `Evict` was introduced.

## Syntax

<TabItem value="evict_1" label="evict_1">
<CodeBlock language="csharp">
{`void Evict<T>(T entity);
`}
</CodeBlock>
</TabItem>

| Parameters | | |
| ------------- | ------------- | ----- |
| **entity** | T | Instance of an entity that will be evicted |

## Example I

<TabItem value="evict_2" label="evict_2">
<CodeBlock language="csharp">
{`Employee employee1 = new Employee
\{
    FirstName = "John",
    LastName = "Doe"
\};

Employee employee2 = new Employee
\{
    FirstName = "Joe",
    LastName = "Shmoe"
\};

session.Store(employee1);
session.Store(employee2);

session.Advanced.Evict(employee1);

session.SaveChanges(); // only 'Joe Shmoe' will be saved
`}
</CodeBlock>
</TabItem>

## Example II

<TabItem value="evict_3" label="evict_3">
<CodeBlock language="csharp">
{`Employee employee = session.Load<Employee>("employees/1-A"); // loading from server
employee = session.Load<Employee>("employees/1-A"); // no server call
session.Advanced.Evict(employee);
employee = session.Load<Employee>("employees/1-A"); // loading from server
`}
</CodeBlock>
</TabItem>
